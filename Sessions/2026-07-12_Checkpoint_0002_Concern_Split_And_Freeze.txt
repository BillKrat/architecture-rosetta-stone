2. Identified the two intertwined concerns in the now-fully-clean map,
   by inspecting every branch's actual content (not just titles):
   - Concern A (the real Rosetta Stone content — Responsibility-Driven
     Design vocabulary for AI systems): Application, Model (+
     API-service/Edge/RAG variants), Agent, Tool, Retrieval, Memory,
     Security, Deployment Context Notes, Cross-cutting fields template,
     Diagnostics/Observability.
   - Concern B (a specific, unrelated implementation project — reads
     like the `vs-mcp-bridge` repo's design, not general AI vocabulary):
     Transport (MCP/stdio/named-pipe), Host (VSIX/IDE), Testing/CI,
     Packaging/Release, Connector/Adapter — their content is explicitly
     VSIX/stdio/MCP-tool-schema specific ("Build VSIX, sign artifacts,"

     tools").
   Proposed this split to Bill for confirmation before cutting anything,
   since it's a structural change to a soon-to-be-frozen document. Bill
   confirmed and specified the mechanism: clone the existing map into a
   new, appropriately-named map, then prune each copy down to only its
   own concern (rather than, e.g., moving nodes piecemeal). Bill also
   clarified that the map's pre-existing empty "Map 2" sheet should NOT
   be used as the Concern B destination — it's earmarked as a
   scratch/playground sheet for the training phase instead.

3. Executed the split in XMind Cloud:
   - Duplicated `rosetta-stone-AI` via the file browser's right-click
     "Duplicate" action, renamed the copy to `rosetta-stone-MCP-Bridge`.
   - In the new `rosetta-stone-MCP-Bridge` copy: deleted every Concern A
     branch (Application, Model, Agent, Tool, Retrieval, Memory,
     Security, Deployment Context Notes, Cross-cutting fields,
     Diagnostics/Observability), leaving only the 5 Concern B branches.
   - In the original `rosetta-stone-AI`: deleted every Concern B branch
     (Transport, Host, Testing/CI, Packaging/Release, Connector/
     Adapter), leaving only the 10 Concern A branches.
   - Verified both results via full page reload + outline text dump
     after each deletion pass — each document contains exactly its own
     concern's branches, nothing more, nothing less, with no title/body
     corruption from the deletion work.
   - Node deletion in the Outliner turned out to be UI-quirky: the
     right-click context menu frequently rendered clipped at the
     browser viewport's bottom edge with no visible "Delete" item and
     no scroll affordance that reliably worked. The dependable method
     that emerged: click the row's drag-handle (the "::::" grip to the
     left of the expand caret) to select the row as a true block (not
     just place a text cursor), then press the actual Delete /
     forward-delete key — this deletes the whole branch cleanly without
     needing the context menu at all. Worth remembering for any future
     node-deletion work in this map.
   - One near-miss during this work: a plain left-click on a row (not
     the drag-handle) plus Backspace was initially mistaken for a
     row-delete shortcut, but it actually entered text-edit mode and
     deleted one character from the title ("paste" -> "pte" in
     "Cross-cutting fields to paste into..."). Caught immediately via
     zoomed screenshot verification, and moot in that instance since
     the row was being deleted outright anyway — but the lesson stands:
     always verify a "delete" action actually removed the row, not just
     that something changed.

4. Renamed `rosetta-stone-AI`, dropping its `[draft]` prefix now that
   it's frozen source of truth rather than a work-in-progress draft.

5. Froze both documents and committed snapshots to this repo as the

   - Downloaded both as native `.xmind` files via each document's
     right-click "Download" action (Bill approved this export approach

     (`*.outline.md`) built from the same verified outline-text captures

     `.xmind` files remain the actual source of record.

METHOD DISCOVERIES
--------------------
- A "fully verified" claim about a large map is only as good as what was
  actually expanded and checked — collapsed branches are invisible to
  both eyeballing and `get_page_text` alike. Any future full-map
  verification pass should explicitly expand every top-level branch
  first, not just the ones already open from prior work.
- XMind Outliner node deletion: use the row's drag-handle to select the
  row as a block, then press Delete/forward-delete. The right-click
  "Delete" menu item is unreliable in this browser context (context
  menu clips at the viewport edge with no reliable way to reach it).

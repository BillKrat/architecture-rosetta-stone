METHOD DISCOVERIES
--------------------
- A search-engine's summarized paraphrase of an authoritative source is
  not the same as the source, even when it sounds right — the ISO/IEC
  22989 "definition" the first search returned was a loose paraphrase,
  not real standard text. Fetching the primary document directly (here,
  the actual NIST PDF) is what `Standards/source-of-truth.md`'s
  `authoritative` bar actually requires; a search result alone doesn't
  clear it.
- U.S. government publications (NIST, in this case) are public domain
  and can be quoted in full; a privately-published book (RDD) cannot be
  treated the same way even when it's the project's own primary
  methodology source and was legitimately purchased — ownership of a
  copy isn't the same as a license to redistribute its text. Worth
  remembering any time a future session cites another purchased/
  licensed source.
- Self-correcting a drafted-but-not-yet-committed mistake (the
  over-quoting) before it reaches git is the same discipline as fixing
  insecure code before shipping it — catch it in the same turn, don't
  let it stand because it was already written.
